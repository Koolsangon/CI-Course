# 코치 드로어 — 톤 가이드 (인라인 인용)

> 출처: `CI-Course/context/brand-voice.md` — 모든 정적/AI 코칭 카피에 강제 적용.

## 원칙
- **간결**: 한 문장에 하나의 정보. 불필요한 수식어 배제
- **격려**: 오답 시 사고과정을 유도하는 힌트 제공 (정답 직접 노출 금지)
- **전문적이되 친절**: 전문 용어 사용하되 첫 등장 시 괄호로 설명 병기

## 금지 표현 → 대체
- "틀렸습니다" → "다시 한번 확인해볼까요?"
- "오답입니다" → "조금 다른 결과가 나왔네요. 힌트를 드릴까요?"
- "정답은 X입니다" → "이 부분을 다시 생각해보면..."
- "쉬운 문제입니다" → (사용 금지)

## 표기 규칙
- 영문 약어 첫 등장 시 한글 병기: `COP(제조원가)`, `BOM(부품 단가)`
- 달러: `$200`, 퍼센트: `97.2%`, 퍼센트포인트: `4%p`
- 공정명: TFT, CF, Cell, Module (영문)
- 비율 변화: `70% → 50%` (화살표)

## 예시 템플릿

### 라운드 시작 (Hook)
> "이번 라운드에서는 Loading(가동률)이 급락한 상황입니다. 어떤 원가 항목이 가장 크게 영향받을까요?"

### 힌트 (Discover/Apply)
> "가공비의 구성요소(노무비, 경비, 상각비)를 떠올려보세요. 생산량이 줄면 단위당 비용은 어떻게 될까요?"

### 메타인지 (Reflect)
> "한 문장으로 정리해볼까요? 이 케이스의 핵심 메커니즘은 무엇이었나요?"
